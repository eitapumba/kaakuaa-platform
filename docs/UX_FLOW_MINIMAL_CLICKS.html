Princípios de UX — Kaa Kuaa
As regras que guiam cada decisão de interface

1. Home = Feed de Desafios

Não existe "dashboard". A home já é o feed de desafios rankeado por AI. O conteúdo mais relevante pro usuário aparece primeiro. Zero navegação necessária.

2. Card = Ação Completa

Cada card de desafio tem TUDO: título, stake, tier, botão de aceitar. O usuário não precisa abrir uma tela separada. Tap → Bottom Sheet → Confirmado.

3. Bottom Sheet, Nunca Nova Tela

Confirmações usam bottom sheet (half-modal). O contexto do feed permanece visível. Menos desorientação, mais velocidade.

4. Biometria > Senha

Auto-login com token persistente. Confirmação de stake com FaceID/TouchID. Nunca pedir senha pra ações do dia-a-dia.

5. AI Feed Ranking

O feed é rankeado considerando: histórico do jogador, categoria preferida, tier, horário, taxa de aceitação, match quality. O melhor desafio sempre aparece primeiro.

6. Primeiro Desafio = Sem Stake

Novos usuários ganham um desafio grátis (Solo, sem dinheiro) para experimentar o fluxo completo antes de comprometer dinheiro. Reduz fricção de ativação.

7. Onboarding = 1 Tela

Interesses + tier de entrada numa única tela. Nada de tutorial de 5 passos. O melhor onboarding é usar o produto.

8. Quick Actions Everywhere

Criar desafio = FAB (floating action button). Investir em player = long press no avatar. Compartilhar = swipe. Ações frequentes são gestos, não menus.

Features que Reduzem Fricção
Coisas que fazem os 2 taps funcionarem na prática

🔐 Pre-Authorized Payments

Na primeira compra o usuário autoriza o Stripe pra cobranças futuras até seu tier máximo. Depois disso, aceitar desafio = biometria, sem tela de pagamento. Similar ao Uber.

📡 WebSocket Real-Time

Quando o 2º jogador aceita, ambos recebem o match via WebSocket instantaneamente. Sem polling, sem refresh. A tela de VS aparece automática.

🤖 Smart Feed Algorithm

O backend rankeia desafios por: matchQuality (skill similar), userPrefs (categorias favoritas), recency (mais novos), engagement (mais assistidos). Personalizado por usuário.

💨 Optimistic UI

Quando o usuário tapa "Aceitar", a UI já mostra "Entrando..." antes da API responder. Se falhar, reverte. Sensação de zero latência.

🔔 Push + Deep Link

Notificação: "🔥 Desafio de CS2 R$300 na sua tier!" → Tap na notificação → App abre DIRETO no card com bottom sheet aberto. 1 tap pra confirmar.

⚡ Instant Match

Se já tem oponente esperando, o POST /join faz match na mesma request. Sem tela de "aguardando oponente". Aceitar = Match imediato quando possível.
